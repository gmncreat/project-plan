표지를 제외한 페이지는 각 번호마다 기술한 내용을 바탕으로 기능 overview 구조도를 표시하고, 개발 내용의 핵심 내용을 텍스트 항목으로 표시한다.
개발 내용에 부족한 부분이 있다면 관련 자료를 조사하여 보충한다.

선로품질관리시스템은 IPOps 시스템의 하위 시스템이고, 기간망 장비 EMS 시스템을 사용하여 OTDR 측정을 관리하는 시스템이다.
선로품질관리시스템은 1차 개발이 완료되었고, 본 문서는 고도화를 위한 개발 계획서이다.

		수집 정보 및 분석된 정보는 OTDR 데이터베이스에 저장한다.

	HUAWEI EMS(NCE: Network Cloud Enging) 인벤토리 정보 조회(REST API 사용)

			NCE에서 SFTP 서버로 파일을 전송하고 이를 수집하여 분석 모듈로 전달하는 도식 표시

3. HUAWEI OTDR 측정 Profile 관리 기능
	개요
		HUAWEI EMS(NCE)는 OTDR 측정에 필요한 Profile 정보를 관리해주는 기능이 없어 선로품질관리시스템에서 OTDR 측정 Profile을 관리하는 기능이다.
		OTDR 측정 Profile 정보는 기간망 네트워크 링에서 link 단위로 관리해야 한다.
		선로품질관리시스템	Web UI는 OTDR 측정 Profile에 대한 생성/조회/수정/삭제 기능을 제공한다.
		OTDR 측정 작업 생성시 이 Profile 정보를 /restconf/v2/operations/ietf-trans-oam:create-otdr-detect-task api를 통해 전달한다.
	
	
	==> HUAWEI OTDR 측정을 위한 Profile 정보외에 더 관리해야 할 정보가 있다면 추전 필요
		
	OTDR 측정 Profile Backend API 모듈
		Web UI에서 사용할 Profile 생성/조회/수정/삭제 화면에서 사용할 API를 제공
		link에 Profile이 없는 경우에 사용할 Default Profile에 대한 조회/수정 API 제공
		
		Profile 정보는 다음 필드 정보를 포함하며, OTDR 데이터베이스에 Profile 테이블에 저장한다.
			refractive-index
			reflection-threshold
			splice-loss-threshold
			fiber-end-threshold
			pulse-width
			distance-range
			scan-time
			detect-mode
	
	OTDR 측정 Profile 관리 화면 기능
		link 탐색
			Link를 검색할 수 있는 필터 항목들
				국사, 링명, 장비명 등
				
		link의 OTDR 측정 Profile 생성/조회/수정/삭제 기능
			Profile 각 필드에 대해 입력가능한 값의 범위 및 종류에 대해 UI에서 상세한 설명 표시

		Default Profile 설정 관리
			Profile을 생성하지 않은 link에 대한 기본 Profile 정보를 관리

	
4. HUAWEI OTDR 측정 작업 실행 기능
	개요
		선로품질관리시스템 UI에서는 다음처럼 측정 종류를 분리하여 OTDR 측정을 실행한다.
			정기 측정: 주기적으로 OTDR 측정
			장애 측정: 장애 발생시 즉시 OTDR 측정
			작업 측정: 장비 작업 전/후 OTDR 측정
			BaseLine 측정: 장비 OTDR Baseline 측정
		
		UI에서 측정을 등록하면 내부 스케줄러에서 스케줄링 알고리즘에 의해 측정 일정을 정한다.
		측정 일정이 trigger 되면 HUAWEI NCE 를 통해 OTDR 측정 요청을 전송한다.
		
	
	OTDR 측정을 위한 구성 정보 조회
		선로품질관리시스템 Web UI에서는 측정 종류와 측정 대상 링을 선택하여 측정 작업을 생성하면,
		OTDR 데이터베이스에서 대상 링과 관련된 모든 link들을 조회하고 관련 장비 및 포트 정보 조회
		link단위 Profile 정보 조회
	
	OTDR 측정 Task 스케줄러 기능
		각 측정 종류 특성에 따라 내부 스케줄러에 의해 각 측정 작업 간 우선순위, 동시 측정 조건 등을 고려하여 측정 작업 일정을 스케줄링함
		
	OTDR 측정 Task 실행 기능
		HUAWEI NCE OTDR 측정 task 요청
	
	측정 Task 관리
		측정 실행 요청한 Task들을 내부적으로 관리하고, task 정보를 NCE에서 주기적으로 polling방식으로 조회하여 task 상태를 관리한다.
		선택 기능
			만약 NCE OTDR 측정 Event를 실시간 공지하는 서비스가 제공된다면, polling이 아닌 OTDR task 관련 event를 연동하여 task 상태를 관리한다.
	
	측정 관련 설정 기능
		HUAWEI NCE OTDR 측정 동시성 파라미터 설정 기능
			OTDR 동시 측정 수 limit
			API 동시 호출 수 limit
			그 외 고려 가능한 동시성 파라미터들 추가
			
		스케줄러 옵션 설정
			정기/장애/작업/Baseline 측정 종류별 실행 설정


5. HUAWEI OTDR 측정 결과 조회 기능
	개요
		OTDR 측정이 완료되면 측정 결과들을 수집 및 분석하여 데이터베이스에 저장한다.
		수집 결과들은 NCE OTDR REST API response 내용, OTDR sor 파일 등이 있다.
		선로품질관리시스템 Web UI에서 측정 결과를 볼 수 있는 화면이 이미 개발되어 있고, 이 화면을 통해 HUAWEI OTDR 측정 결과도 통합하여 조회하고자 한다.
		1차 개발에서 미흡했던 측정 결과 표시 화면들은 개선한다.
	
	
	OTDR 측정 결과 수집
		otdr task rest api resposne 결과 수집
		otdr sor 파일 수집
	
	측정 결과 분석 및 저장
		otdr task rest api resposne 결과 분석하여 데이터베이스에 저장
	
		SOR 파일 분석 기능
			표준 SOR 파일 분석
			데이터베이스 측정 결과 테이블에 저장
		
		==> 그외 더 분석할 측정 결과가 있는경우 추천 필요
	
	
	결과 조회 화면 개선
		측정 결과 표시 화면 고도화


6. HUAWEI EMS 자동점검 조회 기능
	개요
		EMS에서 선로 품질을 관리하기 위해 자동으로 OTDR 측정을 실행한다.
		EMS에서 실행한 OTDR 자동점검 측정 task 이력을 탐색하여 자동점검 측정결과를 수집, 분석, 저장한다.
		선로품질관리시스템 Web UI를 통해 OTDR 자동점검 측정결과를 조회한다.
	
	HUAWEI EMS OTDR 자동점검 측정 발생
		EMS에서 선로 품질을 관리하기 위해 자동으로 OTDR 측정을 실행하고 측정 task 정보를 task 이력에 기록한다.
	
	EMS의 OTDR 자동점검 측정 이력 탐색
		task 이력 주기적으로 polling 조회
			task 이력을 UI에서 설정된 주기에 따라 조회
		
		task 이력 조회 조건 관리
			task 이력 조회 시간범위 정보를 저장하여, 매 탐색 주기마다 이전 주기에서 조회했던 시간 범위 다음부터 조회할 수 있도록 해야 한다.
		
		자동점검 측정 task 판단
			측정 task 이력에는 수동 측정 task 이력과 자동점검 측정 task 이력이 혼재되어 있다

			조회된 task 이력 중에서 자동점검 측정 task인지 판단할 수 있는 정보가 있는 경우 이를 검색하여 자동점검 측정 task를 분류한다.
			만약 자동점검 측정 task인지 판단할 수 있는 정보가 존재하지 않는 경우, 선로품질관리시스템에서 실행한 측정 task 이력을 관리하는 기능이 추가로 필요하다.
				조회된 task 이력에서 수동실행한 task이력을 제외하여 자동점검 task 이력을 추출한다.
	
	EMS의 OTDR 자동점검 측정 이벤트 실시간 연동 방안 연구
		자동점검 측정 이력을 탐색하기 위해 전제 장비를 대상으로 주기적으로 polling 방식으로 조회하는 것은 리소스 부하가 크다
		이를 개선하기 위해 HUAWEI EMS NCE에서 자동점검 측정 이벤트를 실시간으로 연동할 수 있는 방안을 모색한다.
		
		자동점검 측정 이벤트 실시간 연동 기능
			자동점검 측정 이벤트를 실시간 연동할 수 있는 방안이 있는 경우, 실시간 연동 모듈을 개발하여 자동점검 측정 이벤트를 실시간으로 연동한다.
	
	EMS의 자동점검 결과 수집/분석/저장
		자동 점검 측정 task 발견 시 측정 결과를 수집 및 분석하여 데이터베이스 측정결과 테이블에 저장한다.
	
	자동점검 측정결과 조회
		선로품질관리시스템 1차 개발에서 구현된 자동점검 조회 화면을 통해 자동점검 측정 결과를 조회

	자동점검 측정 이력 탐색 설정 관리
		HUAWEI EMS의 OTDR 자동점검 측정 이력 조회 주기 설정


7. TANGO 링 정보, GIS 선번 정보 연동 기능

		연동 대상 정보에 대한 상세 필드 규격을 협의한다.

	연동 대상 정보 데이터 조회 화면 개발

8. NOKIA EMS 자동점검 조회 기능 개발
	개요
		선로품질관리시스템 1차 개발에서는 NOKIA EMS에서 제공하는 OTDR 측정 인터페이스를 구현하고 수동으로 OTDR 측정을 실행할 수 있는 기능을 개발하였다.
		NOKIA EMS는 선로 품질을 관리하기 위해 자동으로 OTDR 측정을 실행한다. 이를 EMS의 OTDR 자동점검이라 한다.
		NOKIA EMS는 하나의 OTDR 측정 실행을 하나의 Job으로 관리하며, 자동점검 측정과 수동 측정을 같은 JobList안에서 관리한다.
		선로품질관리시스템 1차 개발에서는 EMS REST API를 통해 주기적으로 pooling방식으로 전체 JobList를 조회하여 자동점검 측정을 분류한다.
		주기적으로 polling하는 방식은 리소스 부하가 크고 비효율적이다.
		NOKIA EMS에서는 OTDR 관련 이벤트를 포함하여 내부에서 발생하는 이벤트를 Kafka를 통해 연동받을 수 있는 서비스가 존재한다.
		선로품질관리시스템 고도화 프로젝트에서는 EMS 이벤트를 Kafka를 통해 연동받아 자동점검 측정 이벤트가 발생한 경우 이 자동점검 측정결과를 수집, 분석, 저장하고, Web UI에서 자동점검 측정 결과를 조회할 수 있는 기능을 개발한다.

	EMS 이벤트 Kafka 연동 기능 개발
		Kafka SASL_SSL 인증 구현
		이벤트 토픽 연동
		OTDR 자동점검 측정 이벤트 판단
		OTDR 자동점검 측정 결과 수집 모듈로 이벤트 전달
	
	자동점검 측정 결과 수집 모듈 개발
		자동 점검 측정 이벤트를 전달받아 측정 완료 여부 조회
		측정이 완료된 경우 측정 결과 및 SOR 파일 다운로드
		측정 결과 및 SOR 파일 분석하여 데이터베이스 측정결과 테이블에 저장

	자동점검 측정결과 조회
		자동점검 조회 화면을 통해 측정 결과를 조회한다.
	
	자동점검 측정결과 조회 화면 개선
		선로품질관리시스템 1차 개발에서 구현된 자동점검 조회 화면에 대해 사용자 개선 요구사항을 반영하여 측정 결과 조회 화면을 개선한다.


9. OTDR 선로 길이 보정 기능 개발
	개요
		선로품질관리시스템 1차 개발에서 OTDR 측정 결과 데이터를 사용하여 거리에 따른 광신호 세기를 표시하는 OTDR Trace 그래프를 구현하였다.
		그리고 측정 대상 Link 내의 기간망 FDF들간 선로 거리 구성정보를 사용하여 OTDR Trace 그래프에 FDF들의 위치를 표시하였다.
		고도화 프로젝트에서는 발신 장비의 source Port와 첫번째 FDF와의 거리 구성정보를 사용하여 Link 시작 지점에 추가하고, 각 FDF의 위치를 보정한다.
	
	Backend API 수정
		Link의 선로 시작 지점에 발신 장비의 source Port와 첫번째 FDF와의 거리를 추가하고, Link내 각 FDF의 선로상 위치를 추가된 거리만큼 보정하여 계산한다.
		
	OTDR Trace 그래프 수정
		Backend API를 통해 조회한 FDF의 위치 정보를 OTDR Trace 그래프에 표시한다.


10. OTDR 측정 결과 보고서 기능
	개요
		주기적으로 OTDR 측정 결과들을 링단위로 생성한다.
		OTDR 측정 결과를 매 주기마다 생성한다.

		한 주기내에서 실행된 OTDR 측정 결과를 모아 요약하여 보고서를 생성한다.
		보고서 조회 화면에서는 이전 주기에 생성된 보고서 내용과 현 주기에 생성된 보고서 내용과의 차이점을 볼 수 있는 화면을 개발한다.
		보고서 수동 생성 기능은 사용자가 조회하고자 하는 링과 측정 기간을 선택하여 보고서를 수동을 생성할 수 있는 기능을 제공한다.

	보고서 생성 모듈 개발
		입력 파라미터 정보
			보고서 생성 모드
				자동 생성, 수동 생성
			링
			측정 기간
			OTDR 측정 종류
				정기/장애/작업/BaseLine
			비교 대상 보고서 정보
				비교할 측정 기간 또는 비교 대상 보고서 ID
		
		측정 결과 통계 산출
			기간내 OTDR 측정 결과 통계 산출
			통계 대상 및 산출 방식은 사용자와 협의 필요

		보고서 비교
			자동 생성 모드는 이전 주기 보고서와 비교하여 비교 통계 생성
			수동 생성 모드는 비교 대상 보고서와 비교하여 비교 통계 생성
				비교할 측정 기간
					비교할 측정 기간에 대해 보고서를 생성하여 현 보고서와 비교하여 통계 정보 생성
				비교 대상 보고서 ID
					기존 생성된 보고서 ID에 해당하는 보고서 데이터와 현 보고서 비교하여 통계 정보 생성
			비교 대상 정보는 사용자와 협의 필요
		
		보고서 저장
			생성된 보고서 데이터를 데이터베이스 보고서 테이블에 저장
		
	보고서 자동 생성 모듈 개발
		보고서 생성 주기에 따라 모든 링에 대해 보고서 생성 모듈 실행 배치 처리
	
	보고서 Backend API 개발
		정기 보고서 관련 API 개발
		보고서 수동 생성 관련 API 개발
	
	OTDR 측정 결과 보고서 화면 개발
		정기 보고서 조회 화면
			보고서 선택
				조회하려는 링을 선택하고 생성된 정기 보고서 목록에서 하나의 보고서 선택
			보고서 상세 화면
				선택된 보고서 데이터를 보여주는 상세 화면
		
		보고서 수동 생성 화면
			보고서 생성 조건 선택
				링, 측정기간, 측정 종류 등을 선택
			보고서 상세 화면
				생성된 보고서 데이터를 보여주는 상세 화면
		
	OTDR 측정 결과 정기 보고서 설정 관리
		보고서 생성 주기 설정
		보고서 생성 대상 OTDR 측정 종류 설정
			정기/장애/작업/BaseLine
